# Foreign Signer Examples

This directory contains examples of implementing custom cryptographic signers in different programming languages using the `CryptoSigner` trait exposed through UniFFI.

## Overview

The mob library provides a pluggable signing interface that allows you to:

1. **Use the built-in Rust implementation** (`RustSigner`) - A pure Rust implementation using the `k256` and `bip32` crates
2. **Implement custom signers in any supported language** - Python, Swift, Kotlin, Ruby, etc.

## Why Use Custom Signers?

Custom signers allow you to:

- **Use language-native cryptographic libraries** (Python's `cryptography`, Swift's `CryptoKit`, Kotlin's `BouncyCastle`)
- **Integrate with hardware security modules (HSMs)** or hardware wallets
- **Use platform-specific secure enclaves** (Apple Secure Enclave, Android Keystore)
- **Implement biometric authentication** (Touch ID, Face ID, fingerprint)
- **Connect to external signing services** or multi-sig systems
- **Support custom key management strategies**

## The CryptoSigner Trait

All signers must implement the `CryptoSigner` trait defined in Rust:

```rust
pub trait CryptoSigner: Send + Sync {
    /// Get the signer's bech32 address
    fn address(&self) -> String;

    /// Get the signer's compressed secp256k1 public key (33 bytes)
    /// Format: 0x02/0x03 (prefix) + 32-byte X coordinate
    fn public_key(&self) -> Vec<u8>;

    /// Get the address prefix (e.g., "xion")
    fn address_prefix(&self) -> String;

    /// Sign arbitrary bytes with secp256k1 ECDSA
    /// Returns 64-byte signature: r (32 bytes) || s (32 bytes)
    /// Signature MUST be normalized to low-S form
    fn sign_bytes(&self, message: Vec<u8>) -> Result<Vec<u8>, SignerError>;
}
```

## Requirements

Your custom signer implementation MUST:

1. **Return compressed secp256k1 public keys** (33 bytes)
   - First byte: `0x02` (even Y) or `0x03` (odd Y)
   - Remaining 32 bytes: X coordinate

2. **Sign SHA256 prehashed messages**
   - The `message` parameter should be hashed with SHA256 before signing

3. **Return normalized signatures** (64 bytes, low-S form)
   - Format: r (32 bytes) || s (32 bytes) in big-endian
   - Signature MUST be normalized: `s <= curve_order / 2`

4. **Be thread-safe** (Send + Sync in Rust terms)

## Usage Pattern

All examples follow the same basic pattern:

```
1. Import the mob library (generated by UniFFI)
2. Import your language's cryptographic libraries
3. Implement the CryptoSigner trait/interface
4. Create your custom signer instance
5. Pass it to Client.new_with_signer() or SessionSigner
6. Use the client normally - it will call your implementation!
```

## Common Use Cases

### 1. Hardware Wallet Integration

```python
class LedgerSigner(CryptoSigner):
    def __init__(self, ledger_client):
        self.ledger = ledger_client

    def sign_bytes(self, message: bytes) -> bytes:
        # Forward signing request to Ledger device
        return self.ledger.sign(message)
```

### 2. Multi-Signature Coordinator

```kotlin
class MultiSigSigner(private val signers: List<CryptoSigner>) : CryptoSigner {
    override fun signBytes(message: ByteArray): ByteArray {
        // Collect signatures from multiple signers
        val signatures = signers.map { it.signBytes(message) }
        // Combine according to your multi-sig scheme
        return combineSignatures(signatures)
    }
}
```

### 3. Cloud KMS Integration

```swift
class CloudKMSSigner: CryptoSigner {
    private let kmsClient: KMSClient

    func signBytes(message: Data) throws -> Data {
        // Send signing request to cloud KMS
        return try await kmsClient.sign(message: message)
    }
}
```

### 4. Biometric Authentication

```kotlin
class BiometricSigner(
    private val biometricPrompt: BiometricPrompt,
    private val keyAlias: String
) : CryptoSigner {
    override fun signBytes(message: ByteArray): ByteArray {
        // Prompt for biometric authentication
        return biometricPrompt.authenticate()
            .flatMap { loadKeyFromKeystore(keyAlias) }
            .map { key -> key.sign(message) }
            .blockingGet()
    }
}
```

## Building the Examples

### Python

```bash

pip install mob

pip install cryptography bech32
```

## Testing Your Signer

Always test your custom signer implementation thoroughly:

1. **Verify signature format** - Ensure 64-byte output
2. **Test signature normalization** - Verify low-S form
3. **Validate public key format** - Check 33-byte compressed format
4. **Test with known test vectors** - Use standard secp256k1 test cases
5. **Verify address generation** - Compare with known addresses

## Security Considerations

- **Never log or expose private keys**
- **Use secure key storage** (Keychain, Keystore, HSM)
- **Validate all inputs** before signing
- **Implement proper error handling**
- **Use constant-time operations** where possible
- **Follow platform security best practices**
